# Lello Project Instructions

This is the Lello project.

Lello is a reading memory app.

The goal is not just to track books, but to help users retain, revisit, and organize what they read.

Follow the existing template conventions.

Use functional React components.

Use named exports for regular components.

Keep `app/` focused on routing, layout, metadata, and route handlers.

Keep domain UI and logic inside `src/components/[module-name]`.

Keep module hooks inside `src/components/[module-name]/hooks`.

Keep module services inside `src/components/[module-name]/services`.

Keep shared reusable components inside `src/components/shared`.

Keep shadcn primitives inside `src/components/ui`.

Prefer clean, minimal, accessible UI.

Prefer small, incremental changes.

Before coding, inspect existing files and follow current patterns.

Do not add new dependencies unless clearly justified.

Do not implement features outside the current requested scope.

Read these project documents before substantial changes:

- `PRODUCT.md`
- `ROADMAP.md`
- `CONVENTIONS.md`
