# Lello

## Product vision

Lello helps readers remember, reflect on, and revisit what they read.

The product is not primarily a book catalog. Books are containers for the user's own notes, reflections, summaries, and review prompts.

## Problem

Many people finish a chapter or book and feel like they did not retain much. They may remember things only when rereading. Lello helps transform passive reading into active recall and long-term retention.

## Target user

Initial target users:

- people who read non-fiction
- people studying through books
- people who want to retain ideas better
- people who want a personal reading memory system

## MVP scope

The MVP should focus on:

- personal library
- reading queue
- book status
- book details
- notes and reflections
- chapter summaries
- manual written summaries
- future support for dictated summaries
- review prompts
- simple rating

## Out of scope for MVP

Do not implement yet:

- social network
- public profiles
- rankings
- book recommendation algorithm
- marketplace or affiliate links
- complex AI assistant
- mobile app
- advanced analytics
- complex permissions

## Core principle

The value is in the user's input and memory workflow, not in having a perfect external book database.

## UX direction

The UI should feel:

- minimal
- calm
- focused
- modern
- similar in quality to Notion, Readwise, Linear, OpenAI, and Supabase

## Language

User-facing UI should be in Portuguese.
Code, comments, docs, variables, functions, and file names should be in English.
